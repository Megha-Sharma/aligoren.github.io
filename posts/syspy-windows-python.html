syspy Windows Sistem Bilgi Kütüphanesi

2015-04-27 00:08

Comments

Not: Yazı Sonunda Paylaşılan Python Hosted Linki Doxygen ile Hazırlanmıştır. Class Diagramları ve Açıklamaları Görebilirsiniz.

Python ile windows sistem bilgilerini görüntülemek için win32 ya da psutil kütüphanelerinin kurulmuş olması gerekmekte. Ancak bunlarla dahi tam görüntülenemez. Aslında görüntülenir ama sürekli bir uğraş vermek zorunda kalırsınız.

Sürekli bir iterasyon işlemi şart olur. Can sıkıntısına biraz uğraşayım dedim ve sonunda PyPi üzerinde paylaşabileceğim ekstra bir kütüphane ortaya çıktı.

Aslında benim işimi görüyor. Windows kullandığım vakitlerde lazım olan bilgileri almamı sağlıyor.

Kurulum

Kurulum işlemi çok basit. Eğer PyPi kullanarak kurmak isterseniz

pip install syspy

komutunu vermeniz yeterlidir. Kendisi zaten gerekli kurulumu yapacaktır.

Eğer yazı sonunda aşağıda vereceğim test dosyalarının da olduğu repoyu indirip kurulum yapmak isterseniz setup.py dosyasını çalıştırmanız gerekmekte.

python setup.py install

Gereksinimleri de unutmayalım

pip install -r requirements.txt

Bu şekilde gereksinimi yani psutil'i yüklersiniz.

syspy Neler Sunuyor?

CPU, Disk, Information, Memory, Network olmak üzere toplam 5 kategoride özellikler sunmakta. Bunların da alt özellikleri vardır.

CPU:

CPU Times, CPU Percent, CPU Percent Per CPU,
CPU Percent Times, CPU Count, CPU Affinity

Disk:

Disk Partitions, Disk Usage, Disk I/O Counters

Information:

Users, Boot Time, Process Iteration Tool

Memory:

Virtual Memory, Swap Memory, Memory Info,
Memory Info(Extend), Memory Percent, Memory Map

Network:

Network I/O Counters, Net Connections

Sunduğu özellikler şimdilik bunlar. Bu zaten amatör bir çalışma idi. Sunulan özelliklerle ilgili test dosyaları şu hiyerarşiye göre dizilmiştir.

Test Dosyaları:

cpu --
    \test.py
disk --
    \test.py
info --
    \test.py
memory --
    \test.py
network --
    \test.py

Örneklere bakalım:

Bu kadar indirdik kurduk falan. Eğer test dosyalarına bakmadıysanız her kategori için birer örnek vermek gerekir diye düşünüyorum. Artık şöyle böyle import edin demeye gerek duymuyorum. PEP standartlarını okursanız ya da bu konu hakkında döküman bulursanız şöyle yazılıyor: "Bir kütüphanenin aktarım işlemini wildcard ile yapmaktan kaçının. Kütüphane'nin hangi nesnesi ya da işlevi ile çalışmak istiyorsanız direkt olarak onu aktarın."

from syspy import CPU
from syspy import Disk
from syspy import Info
from syspy import Memory
from syspy import Network

cpu = CPU()

print(cpu.getCpuTimes())

disk = Disk()

a,b,c,d,e,f,g = disk.getDiskPartitions()

print("Part Device: {}\nUsage Total: {}\nUsage Used: {}\nUsage Free: {}" \
            "\nUsage Percent: {}\nPart Type: {}\nPart Mountpoint: {}".format(a,b,c,d,e,f,g))

info = Info()

print(info.getBootTime())

memory = Memory()

a,b,c,d,e,f = memory.getSwapMemory()
print("Total: {0} Used: {1} Free: {2} Percent: {3} Sin: {4} Sout: {5}".format(a,b,c,d,e,f))

network = Network()

print(network.getNetConnections())
