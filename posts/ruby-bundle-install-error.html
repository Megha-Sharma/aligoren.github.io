Ruby Bundle Install Error

2015-04-18 00:39

Comments

Ruby'de eklenti kurmanın yolu aynı Python'da olduğu gibi bir paket yöneticisi kullanmaktır. Ancak bunu yaparken dahi hatalar alırız. Örneğin şöyle bir hata

Örneğin Ruby on Rails ile bir proje geliştiriyorsunuz. Bundle ile install diyerek gereksinimleri kurmak isteyeceksiniz. Ama o da ne bir hata aldınız:

C:\RailsWork\> bundle install

Gem::RemoteFetcher::FetchError: SSL_connect returned=1 errno=0 state=SSLv3 
read server certificate B: certificate verify failed 
(https://rubygems.org/gems/FALANCA.gem)
An error occured while installing FALANCA (FALAN_VERSIYON), and Bundler cannot continue.
Make sure that `gem install FALANCA -v 'FALAN_VERSIYON'` succeeds before bundling.

Bu hatayı geçmenizin bir yolu verilen gem linkini direkt olarak kopyalayıp

gem install FALANCA.gem

komutu ile kurmak. Ancak Rails gibi büyük frameworkleri kullanırken bağımlılıklar da çok fazla oluyorlar. Bu duruma kalıcı ve kesin bir çözüm getirmek gerekir.

Bunun içinse şöyle bir şey yapılabilir.

Senaryo 1:

Gemfile dosyanızı düzenleyerek içerisindeki

https://rubygems.org olan adresi http://rubygems.org olarak değiştirmeniz. Tabii bu işlemlerden sonra gem --update system komutunu vermeniz gerekiyor. Ancak yine diğer projelerde hatalar alabilirsiniz. Farklı olarak diğer senaryolara da bakabiliriz.

Senaryo 2:

Öncelikle var olan gem kaynaklarından https:// ile başlayan linki silelim.

gem sources -r https://rubygems.org/ bu komut gem kaynaklarındaki https:// ile başlayan adresi siler. Yerine tekrar ekleme yapalım:

gem sources -a http://rubygems.org/ bu komut ise gem kaynaklarına http:// ile başlayan rubygems adresini ekler. Bu işlemlerden sonra gem update --system komutu ile gem kaynaklarını güncelleyelim.

Tekrardan bundle install komutunu verdiğinizde göreceksiniz ki gereksinimler artık kurulmakta.

İyi çalışmalar.
